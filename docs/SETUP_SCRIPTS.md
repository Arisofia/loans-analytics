# Setup Scripts Reference

**All scripts created for Supabase + Grafana + GitHub integration**

---

## Quick Reference

```bash
# 1. Verify Supabase connection & create .env.monitoring
./scripts/setup/manage-supabase-credentials.sh

# 2. Add credentials to GitHub Actions
./scripts/setup/add-github-secrets.sh

# 3. Setup Grafana with Supabase
./scripts/monitoring/setup-grafana.sh
```

---

## Script Details

### 1. `scripts/setup/manage-supabase-credentials.sh`

**Purpose:** Verify Supabase credentials and setup monitoring configuration

**What it does:**
- ✅ Checks `.env.local` for Supabase credentials
- ✅ Tests REST API connection with curl
- ✅ Validates JWT token format
- ✅ Creates `.env.monitoring` from template
- ✅ Shows GitHub Actions secret commands

**Usage:**
```bash
./scripts/setup/manage-supabase-credentials.sh
```

**Expected Output:**
```
✅ Found SUPABASE_ANON_KEY
✅ Found SUPABASE_SERVICE_ROLE_KEY
✅ Found POSTGRES_PASSWORD
✅ SUPABASE_ANON_KEY is valid JWT token
✅ SUPABASE_SERVICE_ROLE_KEY is valid JWT token
✅ Created .env.monitoring
```

**Environment Variables Read:**
- `SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- `POSTGRES_PASSWORD`

**Files Created/Modified:**
- Creates: `.env.monitoring`
- Reads: `.env.local`

---

### 2. `scripts/setup/add-github-secrets.sh`

**Purpose:** Add Supabase credentials to GitHub Actions secrets

**What it does:**
- ✅ Checks GitHub CLI is installed & authenticated
- ✅ Reads credentials from `.env.local`
- ✅ Adds secrets to GitHub repository
- ✅ Verifies secrets were added

**Requirements:**
- GitHub CLI installed: `brew install gh`
- Authenticated: `gh auth login`

**Usage:**
```bash
./scripts/setup/add-github-secrets.sh
```

**Interactive:**
- Asks for confirmation before adding secrets
- Displays secrets to be added (partial)

**Secrets Added:**
1. `SUPABASE_URL`
2. `SUPABASE_ANON_KEY`
3. `SUPABASE_SERVICE_ROLE_KEY`
4. `SUPABASE_DATABASE_URL`

**Files Read:**
- `.env.local`

---

### 3. `scripts/monitoring/setup-grafana.sh`

**Purpose:** Setup Grafana with Supabase datasources

**What it does:**
- ✅ Creates `.env.monitoring` if missing
- ✅ Validates Supabase API connection
- ✅ Starts Docker Compose monitoring stack
- ✅ Waits for services to start
- ✅ Displays connection info

**Usage:**
```bash
./scripts/monitoring/setup-grafana.sh
```

**Expected Output:**
```
✅ Grafana setup complete!
📊 Access Grafana:
   URL: http://localhost:3001
   Username: admin
   Password: admin123
```

**Services Started:**
- Grafana: `http://localhost:3001`
- Prometheus: `http://localhost:9090`
- AlertManager: `http://localhost:9093`

**Files Used:**
- `.env.monitoring` (created or updated)
- `docker-compose.monitoring.yml`

## Execution Order

### First Time Setup

1. **Verify Credentials**
   ```bash
   ./scripts/setup/manage-supabase-credentials.sh
   ```

2. **Add to GitHub** (Optional, for CI/CD)
   ```bash
   ./scripts/setup/add-github-secrets.sh
   ```

3. **Setup Grafana**
   ```bash
   ./scripts/monitoring/setup-grafana.sh
   ```

### Troubleshooting

If `setup-grafana.sh` fails:
```bash
# Check credentials
./scripts/setup/manage-supabase-credentials.sh

# Check Docker
docker ps
docker logs grafana

# Restart stack
docker-compose -f docker-compose.monitoring.yml restart
```

If GitHub secrets not available in workflows:
```bash
# Verify secrets exist
gh secret list

# Re-add if needed
./scripts/setup/add-github-secrets.sh
```

## Environment Files

### `.env.local` (Your Machine)

**Location:** Project root  
**Git:** ✅ Ignored (in .gitignore)  
**Contains:**
- Supabase credentials
- Database password
- API keys
- JWT secret
- Azure credentials

**How to update:**
```bash
# Edit manually
nano .env.local

# Re-run setup script
./scripts/setup/manage-supabase-credentials.sh
```

### `.env.monitoring` (Grafana Stack)

**Location:** Project root  
**Git:** ✅ Ignored (in .gitignore)  
**Contains:**
- Grafana admin password
- Supabase credentials (same as .env.local)
- Azure subscription ID (optional)

**How to update:**
```bash
# Auto-generated by setup script
./scripts/setup/manage-supabase-credentials.sh

# OR manually
nano .env.monitoring
docker-compose -f docker-compose.monitoring.yml restart
```

## GitHub Actions Secrets

### How Secrets Are Used

**File:** `.github/workflows/tests.yml`

```yaml
integration-tests:
  env:
    SUPABASE_URL: ${{ secrets.SUPABASE_URL }}
    SUPABASE_ANON_KEY: ${{ secrets.SUPABASE_ANON_KEY }}
  run: pytest tests/ -v -m "integration"
```

### View Secrets

**In Terminal:**
```bash
gh secret list
```

**On GitHub Web:**
https://github.com/arisofia/abaco-loans-analytics/settings/secrets/actions

## Credential Rotation

### When credentials expire or need renewal:

1. **Generate new keys in Supabase**
   - https://supabase.com/dashboard/project/goxdevkqozomyhsyxhte/settings/api

2. **Update `.env.local`**
   ```bash
   nano .env.local
   # Update SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY
   ```

3. **Regenerate `.env.monitoring`**
   ```bash
   ./scripts/setup/manage-supabase-credentials.sh
   ```

4. **Update GitHub Secrets**
   ```bash
   ./scripts/setup/add-github-secrets.sh
   ```

5. **Restart services**
   ```bash
   docker-compose -f docker-compose.monitoring.yml restart
   ```

## Cleanup

### Remove Credentials

**From local machine:**
```bash

rm .env.local
rm .env.monitoring
```

**From GitHub:**
```bash
# Navigate to:
# GitHub → Settings → Secrets and variables → Actions
# Delete each secret manually

# OR use GitHub CLI:
gh secret delete SUPABASE_URL
gh secret delete SUPABASE_ANON_KEY
gh secret delete SUPABASE_SERVICE_ROLE_KEY
gh secret delete SUPABASE_DATABASE_URL
```
